# Daily AI & Tech Trends - 2026-03-23

## Pick Up

1. [iPhone 17 Pro Demonstrated Running a 400B LLM](https://twitter.com/anemll/status/2035901335984611412)
   - Why I recommend: On-device inference of a 400B-parameter model on a smartphone is a landmark result for edge ML; it signals that the hardware capability gap between data-center and consumer devices is closing faster than expected, with direct implications for private, low-latency inference pipelines.

2. [Walmart: ChatGPT checkout converted 3x worse than website](https://searchengineland.com/walmart-chatgpt-checkout-converted-worse-472071)
   - Why I recommend: Concrete, production-scale evidence that LLM-based UX can hurt conversion — a high-signal data point for ML engineers evaluating where to deploy and where NOT to deploy generative AI in product funnels.

3. [If DSPy is so great, why isn't anyone using it?](https://skylarbpayne.com/posts/dspy-engineering-patterns/)
   - Why I recommend: A candid analysis of the adoption gap between DSPy's theoretical elegance and real-world usage, with actionable engineering patterns — essential reading for ML practitioners choosing LLM orchestration frameworks.

4. [Autoresearch on an old research idea](https://ykumar.me/blog/eclip-autoresearch/)
   - Why I recommend: A hands-on account of using AI agents to autonomously explore a research hypothesis (ECLIP), surfacing both the power and pitfalls of LLM-assisted research workflows — directly relevant for ML researchers.

5. [I built an AI receptionist for a mechanic shop](https://www.itsthatlady.dev/blog/building-an-ai-receptionist-for-my-brother/)
   - Why I recommend: A detailed end-to-end build of an agentic voice AI in a real SMB setting — valuable for understanding practical constraints (latency, telephony APIs, error handling) that rarely appear in research papers.

6. [KICS GitHub Action Compromised: TeamPCP Strikes Again in Supply Chain Attack](https://www.wiz.io/blog/teampcp-attack-kics-github-action)
   - Why I recommend: The TeamPCP group has now compromised a second widely-used DevSecOps tool (KICS) via GitHub Actions tag poisoning; if your ML CI/CD pipeline uses KICS for IaC security scanning, rotate credentials and audit action pins immediately.

7. [Introducing the Wiz Red Agent - AI-Powered Attacker](https://www.wiz.io/blog/introducing-the-wiz-red-agent)
   - Why I recommend: An LLM-based autonomous red team agent that attacks cloud environments — a concrete demonstration of how AI is shifting offensive security, and a useful benchmark for understanding AI agent capability in adversarial tasks.

8. [Introducing Wiz AI Application Protection Platform (AI-APP)](https://www.wiz.io/blog/introducing-wiz-ai-app)
   - Why I recommend: Wiz's new AI-APP targets the full lifecycle of AI application security (training data, model, runtime), directly addressing risks that ML engineers face when deploying LLMs and agents in production cloud environments.

9. [Trivy under attack again: Widespread GitHub Actions tag compromise secrets](https://socket.dev/blog/trivy-under-attack-again-github-actions-compromise)
   - Why I recommend: Corroborates the Wiz KICS report with additional technical detail on the TeamPCP supply chain campaign — together these paint a clear picture of a coordinated attack on DevSecOps tooling that ML pipelines rely on.

10. [Introducing Wiz Agents & Workflows: Security at the Speed of AI](https://www.wiz.io/blog/introducing-wiz-agents)
    - Why I recommend: Agentic security workflows that can automatically detect, investigate, and remediate cloud security issues are a new paradigm shift — worth understanding both as a security tool and as an architectural reference for building agentic systems.
