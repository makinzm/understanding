## Pick Up

1. [Flash-MoE: Running a 397B Parameter Model on a Laptop](https://github.com/danveloper/flash-moe)
   - Why I recommend: Running a 397B-parameter MoE model on consumer hardware with flash-attention techniques is a practical breakthrough for ML engineers — it opens inference and local experimentation that previously required server clusters.

2. [Trivy Compromised: Everything You Need to Know about the Latest Supply Chain Attack](https://www.wiz.io/blog/trivy-compromised-teampcp-supply-chain-attack)
   - Why I recommend: The "TeamPCP" group injected credential-stealing malware into Aqua Security's widely-used Trivy scanner and related GitHub Actions on 2026-03-19; if your CI/CD pipeline uses Trivy, audit immediately — this is a direct security incident for the ML toolchain.

3. [AI Runtime Threat Detection: From Input to Real-World Impact](https://www.wiz.io/blog/ai-runtime-threat-detection-from-input-to-real-world-impact)
   - Why I recommend: Covers how to detect AI-driven behavior anomalies across model, workload, and cloud layers, which is essential context for anyone deploying ML models in production environments.

4. [OpenClaw Is a Security Nightmare Dressed Up as a Daydream](https://composio.dev/content/openclaw-security-and-vulnerabilities)
   - Why I recommend: A deep-dive security audit of an agentic AI framework that exposes dangerous default configurations — directly relevant for ML engineers evaluating or integrating agent orchestration tools.

5. [Understanding and Reducing AI Risk in Modern Applications](https://www.wiz.io/blog/reducing-ai-risk-across-ai-applications)
   - Why I recommend: Provides a practical taxonomy of AI risk categories (prompt injection, data leakage, model abuse) with concrete mitigation strategies for engineers building LLM-integrated products.

6. [The Future of Version Control](https://bramcohen.com/p/manyana)
   - Why I recommend: A thoughtful critique of Git's limitations and a proposal for a new VCS model — high engagement (277 pts, 157 comments) indicates this touches real pain points for every software engineer.

7. [Would You Click 'Accept'? Automatically detecting malicious Azure OAuth applications using LLMs](https://www.wiz.io/blog/detecting-malicious-oauth-applications)
   - Why I recommend: Using LLMs to classify malicious OAuth app consent prompts is a great example of applying ML to a real-world security problem, with implications for cloud ML platform security.

8. [Introducing AI Cyber Model Arena: A Real-World Benchmark for AI Agents in Cybersecurity](https://www.wiz.io/blog/introducing-ai-cyber-model-arena-a-real-world-benchmark-for-ai-agents-in-cybersec)
   - Why I recommend: A novel benchmark evaluating AI agents on real cybersecurity tasks rather than synthetic problems — important for understanding the current capability frontier of LLM-based agents.

9. [Cloudflare flags archive.today as "C&C/Botnet"; no longer resolves via 1.1.1.2](https://radar.cloudflare.com/domains/domain/archive.today)
   - Why I recommend: Cloudflare's DNS classification of a major archiving service as botnet infrastructure raises significant concerns about DNS censorship and has implications for data collection pipelines that reference archived URLs.

10. [Reports of code's death are greatly exaggerated](https://stevekrouse.com/precision)
    - Why I recommend: A well-argued counterpoint to "AI will replace all coding" narratives, discussing where precision in software engineering still requires human expertise — useful framing for ML engineers thinking about their career trajectory.
