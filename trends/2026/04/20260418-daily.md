4. [Primer on GitHub Actions Security - Threat Model, Attacks and Defenses (Part 1/2)](https://www.wiz.io/blog/github-actions-security-threat-model-and-defenses)
   - Why I recommend: ML engineers routinely wire model training, evaluation, and deployment through GitHub Actions; this deep-dive threat model (poisoned workflows, secret exfiltration, artifact tampering) maps exactly to those pipelines.

5. [Securing AI Applications From Inception to Deployment](https://www.wiz.io/blog/securing-ai-application-from-inception-to-deployment)
   - Why I recommend: Covers the full lifecycle — from prompt injection and data poisoning in development to runtime monitoring in production — providing a practical checklist for teams shipping AI features.

6. [It's OK to compare floating-points for equality](https://lisyarus.github.io/blog/posts/its-ok-to-compare-floating-points-for-equality.html)
   - Why I recommend: Dismantles the reflexive "never compare floats" rule with precise conditions under which exact equality is correct, which matters when writing loss functions, metric thresholds, and determinism checks in ML code.

7. [Michael Rabin has died](https://en.wikipedia.org/wiki/Michael_O._Rabin)
   - Why I recommend: Turing Award winner whose Miller-Rabin primality test and pioneering work on randomized algorithms underpin cryptography and theoretical CS; an important moment to revisit his contributions.

8. [Introducing AI Cyber Model Arena: A Real-World Benchmark for AI Agents in Cybersecurity](https://www.wiz.io/blog/introducing-ai-cyber-model-arena-a-real-world-benchmark-for-ai-agents-in-cybersec)
