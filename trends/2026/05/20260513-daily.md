# Daily AI & Tech Trends - 2026-05-13

## Pick Up

1. [Show HN: Needle: We Distilled Gemini Tool Calling into a 26M Model](https://github.com/cactus-compute/needle)
   - Why I recommend: Distilling a large model's tool-calling capability into a 26M parameter model is a significant efficiency win — this is a concrete example of capability distillation that could be deployed on-device or at the edge, directly relevant for any ML engineer optimizing inference costs.

2. [Fragnesia: Linux Kernel Local Privilege Escalation via ESP-in-TCP](https://www.wiz.io/blog/fragnesia-linux-kernel-local-privilege-escalation-via-esp-in-tcp)
   - Why I recommend: Published today by Wiz Research, this new Linux kernel LPE via ESP encapsulation in TCP is a novel attack vector worth understanding — especially for anyone running ML workloads on Linux servers that could be targeted post-initial-access.

3. [Mini Shai-Hulud Strikes Again: TanStack + more npm Packages Compromised](https://www.wiz.io/blog/mini-shai-hulud-strikes-again-tanstack-more-npm-packages-compromised)
   - Why I recommend: The same supply chain actor that previously targeted SAP npm packages is now hitting TanStack, a widely used React ecosystem library — a reminder that ML/AI projects with JS frontends or build tools are also attack surfaces.

4. [I moved my digital stack to Europe](https://monokai.com/articles/how-i-moved-my-digital-stack-to-europe/)
   - Why I recommend: The highest-engagement HN post today (822 pts, 512 comments) documents a full migration away from US-based cloud services to European alternatives — practical reading for anyone evaluating data sovereignty or compliance requirements.

5. [The US is winning the AI race where it matters most: commercialization](https://avkcode.github.io/blog/us-winning-ai-race.html)
   - Why I recommend: With 352 comments, this post sparked the largest HN discussion of the day; the argument that commercial deployment velocity matters more than benchmark leadership is a useful lens for understanding AI competitive dynamics.

6. [Leaving GitHub for Forgejo](https://jorijn.com/en/blog/leaving-github-for-forgejo/)
   - Why I recommend: 481 upvotes on a detailed account of migrating to self-hosted Forgejo reflects a growing developer sentiment around platform lock-in — relevant context for teams evaluating their own VCS infrastructure decisions.

7. [An idiot's guide to lead optimisation for proteins](https://magnusross.github.io/posts/protein-lead-optimisation-1/)
   - Why I recommend: A practical introduction to ML-driven drug discovery (lead optimisation), bridging chemoinformatics and modern ML techniques — good entry point for ML engineers looking to expand into computational biology.

8. [A Framework for AI Threat Readiness](https://www.wiz.io/blog/ai-threat-readiness-framework)
   - Why I recommend: Wiz's structured framework for assessing and improving an organization's readiness against AI-powered attacks is actionable for security-minded ML practitioners, particularly those deploying models in production cloud environments.

9. [Rars: a Rust RAR implementation, mostly written by LLMs](https://bitplane.net/log/2026/05/rars/)
   - Why I recommend: A candid post-mortem on using LLMs to write a Rust implementation of a real format — the author's observations about what LLMs do well (boilerplate) and poorly (subtle logic) are useful calibration for anyone using AI coding assistants.

10. [Composer leaks contents of tokens configured as GitHub OAuth tokens](https://github.com/composer/composer/security/advisories/GHSA-f9f8-rm49-7jv2)
    - Why I recommend: A security advisory for Composer (PHP package manager) leaking GitHub OAuth tokens in certain error conditions — a concrete supply chain security issue worth knowing about if your ML pipelines use PHP-based tooling or trigger Composer installs.
