# Textbooks Are All You Need

## Overview

This paper introduces **phi-1**, a 1.3B parameter language model for code generation trained on ~7B tokens of high-quality "textbook-style" data. Despite its small size, phi-1 achieves 50.6% pass@1 on HumanEval and 55.5% on MBPP, outperforming much larger models trained on orders-of-magnitude more data (e.g., StarCoder at 15.5B parameters, trained on ~1T tokens, achieves only 33.6% on HumanEval).

The central thesis is that **data quality** — not model or dataset size — is the primary driver of performance. By training on curated, "educationally valuable" code and synthetic textbook content, the model achieves state-of-the-art results at a fraction of the usual compute cost.

> [!NOTE]
> "We use the term 'textbooks' not in the literal sense of classical books used for teaching, but in the spirit of high-quality educational data that is clear, self-contained, instructive, and well-structured."

## Architecture: phi-1

phi-1 is a decoder-only transformer. The full model and its smaller sibling share the same architecture family:

| Parameter | phi-1 (1.3B) | phi-1-small (350M) |
|---|---|---|
| Layers | 24 | 20 |
| Hidden dimension | 2048 | 1024 |
| MLP inner dimension | 8192 | 4096 |
| Attention heads | 32 | 16 |
| Head dimension | 64 | 64 |
| Context length | 2048 | 2048 |

Additional architectural choices:
- **Positional encoding**: Rotary Position Embeddings (RoPE), rotary dimension 32
- **Attention**: FlashAttention for efficient multi-head attention
- **Layer structure**: Parallel MHA-MLP (attention and feed-forward computed in parallel, then summed)
- **Tokenizer**: codegen-350M-mono tokenizer (re-used without changes)
- **Dropout**: Attention dropout and residual dropout both set to 0.1

**Input**: Token sequences of length up to 2048, embedded into $\mathbb{R}^{d}$ where $d = 2048$.
**Output**: Next-token logit distributions over the vocabulary, shaped $\mathbb{R}^{|V|}$.

## Datasets

Training uses approximately **7B tokens** total across three datasets constructed specifically for this work:

### 1. Filtered Code Dataset (~6B tokens)

**Source**: The Stack and StackOverflow (raw code from open-source repositories and Q&A).

**Filtering process**:
1. ~100,000 samples from The Stack and StackOverflow were manually annotated using GPT-4, rated for "educational value for a student learning basic coding concepts."
2. A **random forest classifier** was trained using CodeGen-embedding features to predict educational value.
3. The classifier filtered the full dataset down to ~6B high-quality tokens.

**Impact of filtering**: A 350M model trained on unfiltered data saturated at 12.19% HumanEval even after 200B tokens; the same model trained on the filtered subset reached 17.68%.

### 2. Synthetic Textbook Dataset (<1B tokens)

**Source**: Generated by GPT-3.5 with careful prompting to produce Python-focused textbook chapters.

**Construction**: Prompts were designed to ensure:
- Diversity (randomized topics, target audiences, scenarios)
- Educational clarity (step-by-step explanations, clean code, self-contained examples)
- Coverage of algorithmic reasoning (sorting, searching, graph algorithms, dynamic programming, etc.)

The generated content resembles introductory programming textbooks: narrative explanations followed by illustrative code snippets.

### 3. CodeExercises Dataset (~180M tokens)

**Source**: Also GPT-3.5 generated.

**Construction**: Each example is a Python function signature with a docstring and a complete, correct solution. Topics span basic algorithms through standard library usage (e.g., `collections`, `itertools`).

**Role**: Used exclusively for **finetuning** (not pretraining). The intuition is that exercises directly practice the input-output format of code benchmarks like HumanEval.

### Combined Datasets

| Dataset | Tokens | Used For |
|---|---|---|
| Filtered Code (The Stack + StackOverflow) | ~6B | Pretraining |
| Synthetic Textbooks (GPT-3.5) | <1B | Pretraining |
| CodeExercises (GPT-3.5) | ~180M | Finetuning only |

Filtered Code + Synthetic Textbooks = **CodeTextbook** (pretraining corpus).

## Training Procedure

### Pretraining: phi-1-base

Train a decoder-only transformer on CodeTextbook using next-token prediction.

```
Input:  tokens x_1, ..., x_T   (T ≤ 2048)
Loss:   cross-entropy on x_{t+1} given x_1, ..., x_t
```

Hyperparameters:
- Optimizer: AdamW
- Learning rate schedule: linear warmup → linear decay
- Max LR: 1e-3
- Warmup steps: 750
- Weight decay: 0.1
- Batch size: 1024 sequences
- Total steps: 36,000 (checkpoint taken at 24,000 steps, ~50B tokens seen)
- Precision: fp16
- Hardware: 8× NVIDIA A100 GPUs, under 4 days

### Finetuning: phi-1

Continue training phi-1-base on CodeExercises only.

Hyperparameters:
- Max LR: 1e-4
- Warmup steps: 50
- Weight decay: 0.01
- Batch size: 256 sequences
- Total steps: 6,000
- Duration: ~7 additional hours on the same 8 A100s

## Evaluation

### Standard Benchmarks

- **HumanEval**: 164 hand-written Python programming problems, each with unit tests. Metric: pass@1 (single sample).
- **MBPP**: ~500 crowd-sourced Python problems with test cases. Metric: pass@1.

| Model | Params | Training tokens | HumanEval pass@1 | MBPP pass@1 |
|---|---|---|---|---|
| phi-1 | 1.3B | ~7B | **50.6%** | **55.5%** |
| phi-1-small | 350M | ~7B | 45.0% | — |
| phi-1-base (no finetuning) | 1.3B | ~7B | 29.0% | — |
| StarCoder | 15.5B | ~1T | 33.6% | 52.7% |
| WizardCoder | 15B | — | 57.3% | — |
| GPT-4 | — | — | 67.0% | — |

> [!NOTE]
> WizardCoder (57.3%) uses instruction tuning on top of StarCoder; phi-1 surpasses it with ~10× fewer parameters and far fewer training tokens.

### Unconventional Problems

To reduce contamination risk, the authors created 50 new, original problems not present in any known benchmark. phi-1 achieved **52% understanding score** (GPT-4-graded), close to its HumanEval score.

### Data Contamination Analysis

Two decontamination methods were applied to CodeExercises to assess the risk of benchmark leakage:

1. **Embedding similarity**: Remove CodeExercises samples whose embedding (cosine similarity) to HumanEval/MBPP problems exceeded a threshold.
2. **AST-based similarity**: Remove samples with Abstract Syntax Tree match rate > 0.8 with benchmark problems.

After removing >40% of CodeExercises by these criteria, the retrained phi-1 still outperformed StarCoder, indicating that performance is not explained by memorization.

## Key Findings and Insights

### 1. Data Quality vs. Scale

The filtering experiment (Section 3.1) makes a clean empirical case: the same model trained on filtered data consistently outperforms training on 30× more unfiltered data. This challenges the conventional "more data is better" assumption.

### 2. Finetuning Unlocks Emergent Capabilities

phi-1-base (pretrained, no finetuning) achieves 29% on HumanEval. After finetuning on CodeExercises (which contains simple algorithmic exercises), phi-1 reaches 50.6%—a 21.6 point jump. More surprisingly, finetuning also unlocks:
- Usage of external libraries not in CodeExercises (PyGame, Tkinter, PyTorch)
- Basic chat and instruction-following abilities
- Complex logical reasoning and multi-step planning

The authors hypothesize that "the model has learned general coding structures and problem-solving strategies, not just the specific exercises."

### 3. Scaling Laws Under High-Quality Data

Under standard training data, scaling laws predict roughly logarithmic improvement with compute. The authors argue that textbook-quality data "dramatically changes the shape of scaling laws," enabling smaller models to reach capability thresholds previously associated with much larger systems.

> [!IMPORTANT]
> The paper does not publish the code for generating the synthetic datasets, citing the use of GPT-3.5 under API terms of service. This limits full reproducibility.

## Differences from Similar Approaches

| Aspect | phi-1 | Codex (Chen et al., 2021) | StarCoder (Li et al., 2023) |
|---|---|---|---|
| Model size | 1.3B | 12B | 15.5B |
| Training tokens | ~7B | ~100B | ~1T |
| Data strategy | Quality curation + synthetic textbooks | Raw GitHub code | Filtered GitHub (The Stack) |
| Finetuning | CodeExercises (synthetic) | No separate finetuning | Instruction tuning |
| HumanEval pass@1 | 50.6% | 28.8% | 33.6% |

The key architectural difference is **none** — phi-1 uses a standard decoder-only transformer. The differentiation is entirely in the data pipeline.

> [!TIP]
> For background on the HumanEval benchmark, see the Codex paper: Chen et al. (2021), "Evaluating Large Language Models Trained on Code," https://arxiv.org/abs/2107.03374

## Limitations

- **Python-only**: The model was trained exclusively on Python code and performs poorly on other languages.
- **Prompt sensitivity**: Performance degrades with grammatical errors or unusual prompt formats.
- **Limited scale**: Complex real-world software engineering tasks remain beyond the model's capability.
- **Reproducibility gap**: The synthetic data generation process relies on GPT-3.5 and is not fully open-sourced.
- **Scope**: The paper focuses on a narrow task (code generation) and does not evaluate general language understanding.

## Applicability

**Who**: ML researchers and practitioners who need capable code generation without access to large compute budgets; teams exploring data-centric AI approaches.

**When**: When training data budget (tokens) is constrained but synthetic data generation via LLMs is feasible; when studying the relationship between data quality and model performance.

**Where**: Python code generation tasks, especially HumanEval-style function completion. Not intended for production software engineering or multi-language settings.

# Experiments

- **Dataset**: HumanEval (164 problems), MBPP (~500 problems), 50 custom "unconventional" problems
- **Hardware**: 8× NVIDIA A100 GPUs
- **Optimizer**: AdamW with linear warmup and linear decay
- **Pretraining LR**: 1e-3 (max), 750 warmup steps, 36,000 total steps
- **Finetuning LR**: 1e-4 (max), 50 warmup steps, 6,000 total steps
- **Results**: phi-1 achieves 50.6% HumanEval pass@1 and 55.5% MBPP pass@1 with 1.3B parameters and ~7B training tokens, surpassing StarCoder (15.5B params, ~1T tokens) by 17 percentage points on HumanEval
